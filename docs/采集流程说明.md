# 视频采集流程说明

## 概述

视频采集系统采用**前端轮询驱动模式**，每次请求只处理一页中的一个视频，避免长时间运行导致超时。采集方向从最大页码（最旧视频）递减到 page 1（最新视频）。

## 采集流程步骤

### 步骤 1：触发采集

**接口**: `GET /admin/api/video/collectBySourceId?source_id={id}`

**控制器**: `VideoController::collectBySourceId()`

**服务**: `CollectionTaskService::triggerBySourceId()`

```
1. 检查资源站点是否存在、是否启用
2. 检查是否有运行中的任务（Cache: collection_task_running_{id}）
   - 有 → 直接调用 processNext() 继续处理
   - 无 → 调用 trigger() 启动新任务
```

---

### 步骤 2：启动采集（trigger）

**服务**: `CollectionTaskService::trigger()`

```
1. 检查是否需要断点续采
   - last_collected_vod_id 存在 + last_collected_page > 0 + 2小时内有记录
   → 是：从 lastCollectedPage 开始（断点续采）
   → 否：从 pageCount 开始（首次采集）

2. 如果 pageCount 为空
   → 🌐 远程调用 #1: getVideoList(page=1) 获取总页数
   → 更新数据库 page_count 字段
   → 设置 currentPage = pageCount

3. 🌐 远程调用 #2: getVideoList(page=currentPage)
   → 获取当前页视频列表

4. 如果是断点续采
   → 在当前页查找 last_collected_vod_id
   → 找到：跳过该视频及之前的所有视频
   → 未找到：页码+1向前查找，返回提示信息

5. 缓存视频列表到 Cache
   → collection_video_list_{id} = {list, page, next_page, limit, page_count, type_ids}

6. 标记任务为运行中
   → collection_task_running_{id} = true

7. 更新数据库断点信息
   → last_collected_page = currentPage
   → last_collected_vod_id = ''
   → last_collected_vod_name/pic/year/score/duration

8. 返回采集开始信息
```

---

### 步骤 3：获取视频列表（getVideoList）

**服务**: `AppleCmsService::getVideoList()`

**🌐 远程调用**: `GET {api_url}/api.php/provide/vod?ac=detail&pg={page}&pagesize={limit}`

**请求参数**:
| 参数 | 说明 |
|------|------|
| `ac` | 固定值 `detail` |
| `pg` | 页码（page 1=最新，page N=最旧） |
| `pagesize` | 每页数量（默认20） |
| `type` | 分类ID（可选） |

**响应数据**:
```json
{
  "code": 1,
  "msg": "数据列表",
  "page": 4122,
  "pagecount": 4122,
  "limit": 20,
  "total": 82425,
  "list": [
    {
      "vod_id": "12",
      "vod_name": "视频名称",
      "vod_pic": "封面图URL",
      "vod_year": "2026",
      "vod_score": "8.5",
      "vod_duration": "120",
      "vod_actor": "演员",
      "vod_director": "导演",
      "vod_content": "简介",
      "vod_play_url": "播放地址",
      "vod_play_from": "播放器源",
      "type_name": "分类名称",
      "type_id": "1",
      ...
    }
  ]
}
```

**缓存**: 缓存 key 包含 `typeIds + page + limit`，有效期 1 小时

---

### 步骤 4：处理下一个视频（processNext）

**服务**: `CollectionTaskService::processNext()`

```
1. 从 Cache 获取视频列表
   → collection_video_list_{id}

2. 检查列表是否为空
   → 空：检查是否仍有运行中标记
   → 无标记：返回 idle 状态

3. 检查当前页是否处理完（index >= total）
   → 是：加载下一页
     → 🌐 远程调用: getVideoList(page=next_page)
     → 有数据：更新缓存，重置索引，返回 running 状态
     → 无数据：采集完成，清空断点记录
   → 否：继续处理当前页

4. 获取当前视频 item = list[index]

5. 检查数据完整性
   → vod_play_url、vod_content、vod_pic 是否完整
   → 不完整：🌐 远程调用: getVideoDetail(vod_id) 获取详情

6. 保存视频到数据库（saveVideo）
   → 按 title + release_year 检查是否已存在
   → 不存在：新建视频记录
   → 已存在：比较剧集数量
     → 新剧集数 > 旧剧集数：更新视频信息 + 重新插入剧集
     → 新剧集数 <= 旧剧集数：跳过

7. 批量保存剧集（flushBatchEpisodes）
   → 先删除该视频在该资源站点的旧剧集
   → 再批量插入新剧集

8. 更新索引和断点信息
   → Cache: collection_process_index_{id} = current
   → DB: last_collected_vod_id = vod_id
   → DB: last_collected_vod_name/pic/year/score/duration

9. 返回处理结果（包含 vod_name、current_page 等）
```

---

### 步骤 5：获取视频详情（getVideoDetail）

**服务**: `AppleCmsService::getVideoDetail()`

**🌐 远程调用**: `GET {api_url}/api.php/provide/vod?ac=detail&ids={vod_id}`

**触发条件**: 当 list 中的视频数据不完整时（缺少播放地址、简介或封面）

**响应数据**: 与 getVideoList 相同，但只返回指定 vod_id 的一条视频数据

---

### 步骤 6：保存视频（saveVideo）

**服务**: `AppleCmsService::saveVideo()`

```
1. 按 title + release_year 查询是否已存在

2. 如果不存在
   → 创建新 Video 记录
   → 设置 is_show = 0（默认隐藏）
   → 插入数据库

3. 如果已存在
   → 查询该视频在该资源站点的剧集数量
   → 计算新数据的剧集数量
   → 新剧集数 > 旧剧集数：
     - 更新视频基本信息（封面、简介、演员等）
     - 标记需要更新剧集
   → 新剧集数 <= 旧剧集数：跳过

4. 如果有播放地址
   → 收集到 batchEpisodes 数组
   → 等待 flushBatchEpisodes 批量处理

5. 返回 Video 对象
```

---

### 步骤 7：批量保存剧集（flushBatchEpisodes）

**服务**: `AppleCmsService::flushBatchEpisodes()`

```
1. 删除该视频在该资源站点的所有旧剧集
   → DELETE FROM video_sources WHERE video_id = ? AND source_site_id = ?

2. 遍历 batchEpisodes，解析播放地址
   → 按 $$$ 分隔不同播放器源
   → 按 # 分隔剧集
   → 按 $ 分隔集名和 URL
   → 过滤非视频类型的 URL

3. 批量插入新剧集
   → INSERT INTO video_sources (video_id, source_site_id, name, play_url, sort_order, status)
```

| `type` | 分类ID | 可选 |
| `ids` | vod_id | 获取详情时使用 |

## 断点续采机制

### 存储位置

**collect_sources 表**:
| 字段 | 说明 |
|------|------|
| `last_collected_page` | 最后采集到的页码 |
| `last_collected_vod_id` | 最后采集的视频ID |
| `last_collected_vod_name` | 最后采集的视频名称 |
| `last_collected_vod_pic` | 最后采集的视频封面 |
| `last_collected_vod_year` | 最后采集的视频年份 |
| `last_collected_vod_score` | 最后采集的视频评分 |
| `last_collected_vod_duration` | 最后采集的视频时长 |
| `last_collected_at` | 最后采集时间 |

### 续采流程

```
1. 读取断点信息
2. 从 last_collected_page 开始
3. 🌐 获取该页视频列表
4. 查找 last_collected_vod_id
   → 找到：跳过该视频及之前的视频，继续采集
   → 未找到：页码+1向前查找（资源站更新导致视频后移）
5. 如果到达最大页码仍未找到
   → 清空断点记录，从最大页重新开始
```

### 清空断点时机

- 采集完成（到达 page 1）
- 到达最大页码仍未找到断点视频
- 管理员手动重置采集

## 去重逻辑

### 视频去重

**判断条件**: `title + release_year`

- 同标题 + 同一年份 = 重复视频
- 不同标题或不同年份 = 不同视频

### 剧集更新

**判断条件**: 剧集数量

- 新剧集数 > 旧剧集数 → 删除旧剧集，插入新剧集
- 新剧集数 <= 旧剧集数 → 跳过

## 缓存机制

| 缓存 Key | 内容 | 有效期 |
|----------|------|--------|
| `collection_video_list_{id}` | 当前页视频列表 | 1小时 |
| `collection_process_index_{id}` | 当前处理索引 | 1小时 |
| `collection_progress_{id}` | 采集进度 | 1小时 |
| `collection_task_running_{id}` | 运行中标记 | 1小时 |
| `collection_task_last_run` | 上次采集时间 | 24小时 |
